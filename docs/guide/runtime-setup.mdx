Use this page after the first setup is already working and you want to tune SimpleX runtime storage or split multiple SimpleX identities across accounts. For the first install and pairing flow, start with [Getting Started](/guide/getting-started).

## Node runtime storage

This plugin uses the official <code>simplex-chat</code> Node package in the OpenClaw process. There is no separate <code>simplex-chat</code> CLI process and no local WebSocket endpoint.

```json
{
  "channels": {
    "openclaw-simplex": {
      "enabled": true,
      "connection": {
        "dbFilePrefix": "~/.openclaw/simplex/openclaw-simplex"
      }
    }
  }
}
```

The default database file prefix is `~/.openclaw/simplex/openclaw-simplex`. For named accounts, the default suffix is the account id, for example `~/.openclaw/simplex/openclaw-simplex-ops`.

## Multiple accounts

Give each account its own database file prefix so each SimpleX identity has isolated local state.

```json
{
  "channels": {
    "openclaw-simplex": {
      "enabled": true,
      "accounts": {
        "ops": {
          "enabled": true,
          "connection": {
            "dbFilePrefix": "~/.openclaw/simplex/openclaw-simplex-ops",
            "displayName": "OpenClaw Ops"
          }
        },
        "support": {
          "enabled": true,
          "connection": {
            "dbFilePrefix": "~/.openclaw/simplex/openclaw-simplex-support",
            "displayName": "OpenClaw Support"
          }
        }
      }
    }
  }
}
```

## Runtime fields

- <code>connection.dbFilePrefix</code>: SQLite database file prefix used by the SimpleX Node runtime.
- <code>connection.displayName</code>: display name used when the runtime creates a SimpleX profile.
- <code>connection.fullName</code>: full name used when the runtime creates a SimpleX profile.
- <code>connection.migrationConfirmation</code>: database migration behavior, one of <code>yesUp</code>, <code>yesUpDown</code>, <code>console</code>, or <code>error</code>.
- <code>connection.connectTimeoutMs</code>: runtime connection/start timeout.
- <code>connection.autoAcceptFiles</code>: whether incoming file transfers should be auto-accepted.

## Operational notes

The SimpleX runtime shares process fate with OpenClaw. That is the intended official integration path and avoids a separate unauthenticated WebSocket control surface. Treat the database files and file-transfer directories as sensitive local state.
